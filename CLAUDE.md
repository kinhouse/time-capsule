# Development Guidelines

## TDD

Use strict Red/Green/Refactor TDD for all development work:

1. **Write a failing test first.** Confirm it fails for the right reason before writing any production code.
2. **Write only enough production code to make the test pass.**
3. **Refactor** with tests staying green.

Never write production code without a preceding failing test.

## Decision-making

Make implementation decisions independently. Ask the human before making decisions that affect UX.
